# Code of Conduct

## Our commitment

AdByG0d is a security assessment platform maintained by [White0xdi3](https://github.com/White0xdi3). We are committed to keeping this project's community — contributors, issue reporters, and reviewers — professional, respectful, and focused on the work.

Everyone who participates in this project is expected to treat others with dignity, engage substantively with the technical work, and keep all interactions free from harassment, discrimination, and personal attacks.

## Scope

This Code of Conduct applies to all project spaces: the GitHub repository, issues, pull requests, discussions, code review comments, and any other venue where someone is representing or participating in the AdByG0d project.

## Expected behavior

- Communicate respectfully and professionally
- Focus feedback on the technical work, not the person
- Accept constructive criticism gracefully
- Be specific and factual when reporting bugs, proposing changes, or giving review feedback
- Use only synthetic or authorized data in all examples, issues, and contributions — never post real client data, credentials, hashes, or engagement evidence
- Keep discussion on topic and proportionate to the issue at hand

## Unacceptable behavior

- Harassment, intimidation, or personal attacks in any form
- Discriminatory language or behavior
- Publishing private information about others without consent
- Posting real credentials, client data, domain captures, hashes from real environments, private keys, or any evidence from unauthorized or actual engagements
- Deliberate disruption of discussions or project infrastructure
- Misrepresenting your identity, affiliation, or the purpose of a contribution

## Security issues

Security vulnerabilities must not be reported in public issues. Follow the process in [SECURITY.md](SECURITY.md). Posting vulnerability details publicly before coordinated disclosure is a violation of this Code of Conduct and may cause real harm.

## Enforcement

Violations may result in:
1. A written warning with a request to correct the behavior
2. Temporary restriction from contributing to project spaces
3. Permanent ban from the project for severe or repeated violations

The project maintainer ([White0xdi3](https://github.com/White0xdi3)) makes enforcement decisions. If you believe a decision was made in error, you may contact the maintainer directly with a private explanation.

## Reporting

To report a violation, contact the maintainer privately via GitHub. Include a factual description of what happened, links or screenshots where applicable, and the names of anyone involved. All reports are handled confidentially.
