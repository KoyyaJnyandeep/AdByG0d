# Security Policy

## Supported versions

Latest `main` branch and the most recent tagged release.

## Reporting a vulnerability

Do not open public issues for security vulnerabilities.

Report privately through GitHub Security Advisories, or contact the maintainer listed in the repository profile. Include:

- Affected version or commit
- Steps to reproduce using a local or synthetic environment
- Impact and affected component
- Suggested fix if you have one

Do not attach real client data, credentials, private keys, hashes, or raw engagement output.

## Disclosure timeline

Give maintainers a reasonable window to investigate and ship a fix before public disclosure. We aim to acknowledge reports within 72 hours and resolve critical issues within 14 days.

## Authorized use

AdByG0d is assessment tooling for environments you own or have explicit written permission to test. It must not be used against systems without authorization. Users are responsible for legal compliance and evidence handling.
